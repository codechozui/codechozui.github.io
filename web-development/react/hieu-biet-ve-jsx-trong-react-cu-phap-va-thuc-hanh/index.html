Hiểu biết về JSX trong React: Cú pháp và Thực hành

Giới thiệu

Khi bạn học React, việc nắm vững JSX là rất quan trọng. JSX là một tính năng mạnh mẽ của React cho phép bạn viết cú pháp giống như HTML trong JavaScript, giúp cho các thành phần React trở nên dễ đọc và dễ bảo trì hơn.

JSX là gì?

JSX (JavaScript XML) là một phần mở rộng cú pháp cho JavaScript cho phép bạn viết HTML bên trong JavaScript. Điều này giúp mã sạch hơn so với việc sử dụng React.createElement().

Ví dụ không sử dụng JSX:

return (
  React.createElement("h1", {}, "Hello, World!");
);

Ví dụ với JSX:

return <h1>Hello, World!</h1>;

Như bạn thấy, JSX làm cho mã trở nên sạch và dễ đọc hơn.

Những quy tắc quan trọng của JSX

Để viết JSX tốt hơn, hãy nhớ những quy tắc sau:

1. Cần phải có một phần tử cha

Bạn phải bọc nhiều phần tử bên trong một phần tử cha duy nhất.

Sai:

return (
  <h1>Hello</h1>
  <p>Welcome to React!</p>
);

Đúng:

return (
  <div>
    <h1>Hello</h1>
    <p>Welcome to React!</p>
  </div>
);
2. Viết biểu thức JavaScript bên trong {}

Nếu bạn muốn viết bất kỳ biểu thức nào của JavaScript trong JSX, hãy bọc biểu thức JavaScript bên trong {}.

Ví dụ:

const name = "Shefali";
return <h1>Hello, {name}!</h1>;
3. Tên thuộc tính phải sử dụng camelCase

Trong JSX, bạn không thể viết thuộc tính theo kiểu HTML. Thay vào đó, bạn phải sử dụng định dạng camelCase.

Ví dụ: Thay vì viết onclick, hãy viết onClick.

4. Styles inline là các đối tượng JavaScript

Trong JSX, thuộc tính style sử dụng một đối tượng với các thuộc tính camelCase.

Ví dụ:

const styleObj = { color: "blue", fontSize: "20px" };
return <p style={styleObj}>Styled Text</p>;
5. Rendering có điều kiện trong JSX

Sử dụng toán tử điều kiện hoặc && cho các điều kiện.

Ví dụ với toán tử điều kiện:

const isLoggedIn = true;
return <h1>{isLoggedIn ? "Welcome Back!" : "Please Sign In"}</h1>;
6. Rendering danh sách trong JSX

Sử dụng .map() để render danh sách và luôn cung cấp một thuộc tính key.

Ví dụ:

const names = ["Alice", "Bob", "Charlie"];
return (
  <ul>
    {names.map((name) => (
      <li key={name}>{name}</li>
    ))}
  </ul>
);
7. Thuộc tính class là className trong JSX

Trong HTML, bạn sử dụng class, nhưng trong JSX, bạn dùng className.

Ví dụ:

return <h1 className="heading">Hello, JSX!</h1>;
8. Các thẻ tự đóng phải có /

Các phần tử tự đóng như <img> và <input> phải có một / trong JSX.

Ví dụ:

return <img src="image.jpg" alt="Example" />;

Quy trình biên dịch JSX

JSX không chạy trực tiếp trong trình duyệt. Trình biên dịch Babel chuyển đổi mã JSX thành React.createElement().

Mã JSX:

return <h1>Hello, JSX!</h1>;

Babel chuyển đổi như sau:

return (
  React.createElement("h1", {}, "Hello, JSX!");
);

Điều này chứng minh rằng JSX chỉ là một cú pháp dễ hơn, giúp viết mã React dễ dàng hơn.

Kết luận

Chúng ta đã điểm qua nhiều khía cạnh của JSX và cách sử dụng nó trong các thành phần chức năng. Hãy áp dụng những kiến thức này vào dự án của bạn để viết mã sạch hơn và hiệu quả hơn nhé! Nếu bạn có bất kỳ câu hỏi nào, đừng ngần ngại chia sẻ.

Chúc bạn học tập vui vẻ và tiếp tục lập trình!

Hướng dẫn thiết lập WAMP stack với nhiều phiên bản PHP trên Windows

Giới thiệu

Mình đã từng thắc mắc rằng làm thế nào để gỡ bỏ WampServer, Xampp, EasyPHP và các công cụ tương tự? Trong bài viết này, mình sẽ hướng dẫn các bạn:

  • Cách thiết lập PHP, MySQL/MariaDB và Apache trên máy phát triển Windows (WAMP stack)
  • Cách chạy nhiều phiên bản PHP trên máy phát triển mà không cần chuyển đổi liên tục
  • Cách thiết lập vhosts và batch scripts để thực hiện điều đó một cách hiệu quả

💡 Bài viết này dành cho các lập trình viên PHP trung cấp hoặc những ai đang tìm kiếm cấu hình này. Đừng lo lắng nếu các phiên bản PHP khác nhau không có ý nghĩa với bạn, hoặc bạn không chắc chắn về khái niệm server, hãy bookmark bài viết này và quay lại sau.

Trong những năm đầu lập trình, mình đã sử dụng nhiều công cụ như WampServer, XAMPP hay EasyPHP cho phát triển PHP. Mình nhanh chóng nhận ra rằng các công cụ này có một nhược điểm chung: Chu kỳ phát hành chậm và tính linh hoạt hạn chế.

Chúng cho phép mình chuyển đổi phiên bản PHP, nhưng không cho phép chạy nhiều phiên bản PHP cùng lúc, vì vậy mình phải liên tục chuyển đổi phiên bản khi làm việc trên các dự án khác nhau.

WAMP stack

WAMP stack bao gồm 3 thành phần chính mà mình cần cài đặt và cấu hình:

  • Apache server (sử dụng vhosts)
  • MySQL / MariaDB database
  • PHP

Lưu ý rằng W đứng cho Windows, nên nó được gọi là “WAMP”.

Mình có thể cài đặt nhiều phiên bản PHP trên một máy phát triển Windows. Một phiên bản MySQL hoặc MariaDB là tất cả những gì mình cần, nhưng nếu cần, mình có thể chạy các phiên bản khác nhau trên các cổng khác nhau.

Thiết lập & Cài đặt

Mình thích tạo một thư mục dành riêng cho các công cụ phát triển của mình, chẳng hạn như C:\dev. Mình sẽ gọi đây là “thư mục cài đặt”. Sau đó, mình giữ các dự án trong một thư mục riêng, gọi là C:\projects hoặc “thư mục dự án”.

Trên thực tế, mình giữ các dự án trên một ổ đĩa mã hóa riêng biệt, nhưng điều đó không nằm trong phạm vi của bài viết này.

Hãy chọn phiên bản x64 của phần mềm nếu máy của bạn là x64 (thực tế là trường hợp phổ biến). Đừng kết hợp x64 với x32 vì một số kết hợp có thể không hoạt động.

Mẹo để sử dụng nhiều phiên bản nằm ở hai phần:

  • Cấu hình vhosts cho từng dự án (chỉ định directive FCGIWrapper với phiên bản PHP của bạn)
  • Thiết lập các lệnh batch alias (.bat) để gọi các executable PHP (và composer) khác nhau, ví dụ: php84, php82, php74, composer@84, v.v.

PHP

Tải về PHP Windows binaries mà bạn chọn. Hãy chọn phiên bản mới nhất, nếu có thể.

Có hai loại build mà bạn sẽ thấy: Thread safe (còn gọi là TS hoặc ZTS) và Non thread safe (còn gọi là NTS). Mình sẽ sử dụng phiên bản NTS vì nó hoạt động tốt hơn.

💡 Lưu ý rằng với một thiết lập Apache + PHP “typical” với mod_php, như trong XAMPP, bạn sẽ phải chọn phiên bản thread safe (ZTS) để tránh các vấn đề về đồng thời.

Giải nén nội dung vào thư mục cài đặt của bạn, đảm bảo tạo một thư mục con cho mỗi phiên bản PHP chính, như php84, php82, php74… Để đảm bảo nhiều phiên bản có thể tồn tại bên cạnh nhau.

Đường dẫn tới PHP của bạn sẽ trông giống như sau C:\dev\php\php84.

Để cấu hình PHP, hãy tìm tệp php.ini. Nếu không có, hãy tìm php.ini-development, sao chép nó và đổi tên thành php.ini.

Mình sẽ để việc cấu hình thực tế cho bạn. Nhưng mình khuyên bạn nên điều chỉnh các giá trị mà bạn thấy không phù hợp sau này.

Bạn có thể thử nghiệm hoặc xác minh cấu hình bằng cách gọi php -v để xem phiên bản, php -m để xem các mô-đun đã cài đặt hoặc php -i để xem tất cả các cấu hình, điều này khá chi tiết. Hãy chú ý đến bất kỳ thông báo lỗi/cảnh báo nào được PHP in ra.

Để thử nghiệm giao diện web, hãy tạo một tệp đơn giản info.php và đặt lệnh gọi phpinfo ở đó.

<?php phpinfo();

Sau đó, truy cập nó trên trình duyệt thông qua miền cụ thể (xem vhosts bên dưới). Bạn sẽ thấy một giao diện như thế này:

HTTPS (curl, OpenSSL)

Để có thể thực hiện các yêu cầu HTTPS từ PHP một cách chính xác, hãy truy cập CA certificates extracted from Mozilla và tải về tệp cacert.pem. Đặt nó vào thư mục cài đặt của bạn.

Hãy chắc chắn bao gồm nó trong tệp php.ini của bạn (cập nhật các giá trị hiện có):

[curl]
curl.cainfo = "C:/dev/cacert.pem"

[openssl]
openssl.cafile= "C:/dev/cacert.pem"

Xdebug

Truy cập Xdebug downloads và chọn phiên bản phù hợp. Hãy đảm bảo chọn phiên bản PHP đúng + x64 + biến thể NTS.

Giải nén tệp zip - bạn chỉ cần tệp php_xdebug-3.4xxxxx.dll. Đặt dll vào thư mục ext của PHP.

Bao gồm hoặc sửa đổi phần Xdebug trong tệp php.ini.

✋ Hãy cẩn thận ở đây khi chỉ định zend_extension không có tiền tố php_ và hậu tố .dll, nếu không mô-đun sẽ không được tải. Giá trị thực tế ở đây phụ thuộc vào tên tệp mà bạn đặt trong thư mục ext.

[Xdebug]
zend_extension=xdebug-3.4.1-8.4-nts-vs17-x86_64

; Debug enables step debugging, develop has better performance.
; https://xdebug.org/docs/all_settings#mode
xdebug.mode=debug

Bạn có thể đổi tên tệp thành php_xdebug.dll và sử dụng zend_extension=xdebug trong tệp ini, nhưng mình khuyên bạn nên giữ nguyên tên tệp gốc. Nếu bạn quyết định đổi tên, bạn phải giữ tiền tố php_ trong tên tệp hoặc mô-đun sẽ không được tải.

Khi đã cấu hình đúng, gõ php -v trong terminal sẽ xác nhận cấu hình chính xác với Xdebug. Nếu có bất kỳ vấn đề gì, PHP sẽ in chúng ra.

Cập nhật

Để cập nhật một phiên bản PHP, chỉ cần tải về phiên bản mới và ghi đè nội dung thư mục cài đặt. Điều này sẽ giữ lại bất kỳ mở rộng nào bạn đã thêm (như Xdebug) cũng như tệp cấu hình (php.ini) không bị thay đổi.

Apache server

Apache web server là phần mềm lắng nghe các yêu cầu HTTP từ khách hàng (tức là trình duyệt), xử lý các yêu cầu này bằng cách gọi dịch vụ trình thông dịch PHP và gửi phản hồi kết quả về cho người dùng.

Mình sẽ sử dụng module có tên FastCGI (mod_fcgid) với các phiên bản PHP NTS.

Truy cập Apache Lounge download page và

  • Tải về Apache 2.4 x64
  • Tải về module FastCGI (mod_fcgid)

Giải nén nội dung vào thư mục cài đặt của bạn, ví dụ: C:\dev\Apache24, và đặt mod_fcgid vào thư mục \modules, ví dụ: C:\dev\Apache24\modules.

Hãy chắc chắn kích hoạt module trong httpd.conf nằm trong C:\dev\Apache24\conf:

LoadModule fcgid_module modules/mod_fcgid.so

Mình thường tách cấu hình Apache theo từng dự án vào một vài tệp cấu hình nằm trong thư mục dự án, có thể được bao gồm từ httpd.conf bằng một dòng như thế này (đặt ở cuối):

IncludeOptional "R:\httpd-include.conf"

Đây là cấu hình DEV hiện tại của mình, bạn có thể điều chỉnh cho phù hợp hoặc không bao gồm nó để sử dụng mặc định. Mình đã sử dụng cấu hình này khi thử nghiệm với SSE (server-sent events), nhưng sau đó mình đã quên mất. Có thể nó là một sự thừa thãi.

<IfModule fcgid_module>
    FcgidIOTimeout 600
    FcgidConnectTimeout 16
    FcgidBusyTimeout 600
    FcgidMaxRequestsPerProcess 500
    FcgidMaxProcesses 50
    # 4G = 4294967296
    FcgidMaxRequestLen 4294967296

    # cho các thử nghiệm với SSE / streaming
    FcgidOutputBufferSize 0
</IfModule>

Apache sẽ được cài đặt như một dịch vụ. Bạn có thể muốn cấu hình nó không tự động khởi động và khởi động thủ công (nhấn nút Start và gõ “services”). Bạn có thể sử dụng “apache monitor” đi kèm với server. Ứng dụng này cho phép bạn khởi động/dừng và theo dõi Apache từ thanh tác vụ.

Tuy nhiên, mình sử dụng một script nhỏ để khởi động/dừng Apache, cơ sở dữ liệu và Garnet chỉ với một lệnh. Chi tiết hơn về điều này sẽ có sau.

Bạn có thể kiểm tra cấu hình của Apache server bằng cách gọi apache với tham số -t: httpd.exe -t.

Vhosts (virtual hosts)

“vhost” (viết tắt của “virtual host”) là một cấu hình cách ly lắng nghe yêu cầu trên một giao diện / miền nhất định.

Mình đặt các cấu hình vhost vào tệp cấu hình R:\httpd-include.conf được bao gồm từ tệp cấu hình chính. Mình sử dụng một vài tệp bao gồm như vậy, nhưng việc tổ chức là tùy thuộc vào bạn.

Mình sử dụng một vhost và miền riêng cho mỗi dự án mình làm việc để linh hoạt, từ việc tự động điền và những lợi ích khác.

Dưới đây là mẫu cấu hình đơn giản cho một vhost:

<VirtualHost *:80>
    ServerName your-project.lo
    DocumentRoot "R:/your-project"
    <directory "R:/your-project">
        Options +ExecCGI +Indexes +FollowSymLinks
        AddHandler fcgid-script .php
        FCGIWrapper "C:/dev/php/php84/php-cgi.exe -c C:/dev/php/php84/php.ini" .php
        CGIPassAuth on
        AllowOverride all
        Require all granted
    </directory>
</VirtualHost>

👉 Để chuyển đổi phiên bản PHP, tất cả những gì cần thay đổi là directive FCGIWrapper. 👈

Vhost được thiết lập một lần, vì vậy không cần phải chuyển đổi gì sau đó.

Tùy thuộc vào cấu hình của bạn, nhiều vhosts với các phiên bản PHP khác nhau có thể trỏ đến cùng một thư mục (nếu chuyển đổi git branch là cách bạn làm việc khi nâng cấp).

Mình sử dụng .lo làm TLD, nhưng cũng thấy .loc và .local. Bạn có thể thử nghiệm ở đây.

Bạn cũng có thể định nghĩa một server alias, chẳng hạn như www.your-project.lo bằng cách thêm ServerAlias www.your-project.lo vào cấu hình.

Mình sử dụng điều này như một mẫu để tạo các vhosts khác, cũng có thể tự động hóa việc tạo ra chúng, nhưng thực tế không tốn nhiều công sức.

Tệp hosts

Bạn có thể đã kiểm tra rằng khi truy cập your-project.lo trên trình duyệt, nó có thể chuyển đến công cụ tìm kiếm của bạn.

Máy tính của bạn không nhận diện miền và tìm kiếm địa chỉ IP bằng DNS, điều này sẽ thất bại với TLD .lo và bạn sẽ quay lại trang tìm kiếm.

Để khắc phục điều này, hãy chỉnh sửa tệp hosts trên hệ thống của bạn. Tệp này luôn được kiểm tra trước bất kỳ yêu cầu DNS nào được phát ra.

Tệp nằm ở đây:

C:\Windows\system32\drivers\etc\hosts

Tệp này có thể được sử dụng cho nhiều mục đích khác, nhưng ở đây mình sẽ chỉ định các miền phát triển của mình đến localhost như sau:

#
# Dự án của tôi
#
  127.0.0.1      your-project.lo
  127.0.0.1      adminer.lo next.adminer.lo
#...

Điều này sẽ chỉ cho hệ thống biết hướng tất cả lưu lượng mạng đi đến your-project.lo, adminer.lo và next.adminer.lo đến localhost của bạn. Apache sẽ xử lý từ đó.

Lưu lại tệp này và bạn đã sẵn sàng. Sau khi truy cập miền dự án của bạn trong trình duyệt, bạn nên thấy nó được hiển thị.

Tệp này có thể là tệp chỉ đọc hoặc được bảo vệ hệ thống, vì vậy hãy tìm kiếm lỗi ghi khi bạn lưu nó.

Lưu ý rằng để chỉnh sửa tệp hosts, bạn sẽ cần mở nó với quyền quản trị, nếu bạn gặp lỗi khi lưu các thay đổi. Hoặc bạn có thể thử Microsoft PowerToys đi kèm với trình chỉnh sửa hosts và một số công cụ hữu ích khác.

Vhost với ví dụ HTTPS

Ở đây, mình sẽ thiết lập các miền adminer.lo và next.adminer.lo cho phát triển cục bộ với chuyển hướng HTTPS.

Mình tin rằng để sử dụng HTTPS, bạn sẽ phải tạo chứng chỉ tự ký của riêng mình. Mình đã làm điều này từ lâu và chưa tài liệu hóa cách thực hiện, vì vậy mình sẽ để bạn tự thử nghiệm. Hoặc sử dụng AI yêu thích của bạn để gợi ý.

Dưới đây là cấu hình vhost cho adminer.lo chạy trên PHP 7.4:

<VirtualHost *:80>
    ServerName adminer.lo
    Redirect / https://adminer.lo/
</VirtualHost>
<VirtualHost 127.0.0.1:443>
    ServerName adminer.lo
    DocumentRoot "R:/adminer"
    ErrorLog "R:/adminer/error.log"
    <directory "R:/adminer">
        Options +ExecCGI +Indexes +FollowSymLinks
        AddHandler fcgid-script .php
        FCGIWrapper "C:/dev/php/php74/php-cgi.exe -c C:/dev/php/php74/php.ini" .php
        CGIPassAuth on
        AllowOverride all
        Require all granted
    </directory>
    SSLEngine on
    SSLCertificateFile ${SRVROOT}/conf/server.crt
    SSLCertificateKeyFile ${SRVROOT}/conf/server.key
</VirtualHost>

Và đây là vhost cho phiên bản Adminer đã cập nhật của mình:

<VirtualHost *:80>
    ServerName next.adminer.lo
    Redirect / https://next.adminer.lo/
</VirtualHost>
<VirtualHost 127.0.0.1:443>
    ServerName next.adminer.lo
    DocumentRoot "R:/adminer-next"
    ErrorLog "R:/adminer-next/error.log"
    <directory "R:/adminer-next">
        Options +ExecCGI +Indexes +FollowSymLinks
        AddHandler fcgid-script .php
        FCGIWrapper "C:/dev/php/php84/php-cgi.exe -c C:/dev/php/php84/php.ini" .php
        CGIPassAuth on
        AllowOverride all
        Require all granted
    </directory>
    SSLEngine on
    SSLCertificateFile ${SRVROOT}/conf/server.crt
    SSLCertificateKeyFile ${SRVROOT}/conf/server.key
</VirtualHost>

Bất cứ khi nào mình truy cập https://adminer.lo/ trong trình duyệt, PHP 7.4 sẽ phục vụ yêu cầu này; nhưng khi mình truy cập https://next.adminer.lo/, PHP 8.4 sẽ phục vụ yêu cầu đó.

Mình không cần phải chuyển đổi gì và mình có thể làm việc trên cả hai song song.

Document roots riêng cho các phiên bản PHP khác nhau

Ở trên, mình đã trình bày cách cấu hình một vhost/mien cho mỗi dự án. Nếu bạn muốn sử dụng một vhost duy nhất cho một phiên bản PHP, điều đó cũng có thể. Bạn có thể chọn sử dụng các miền như php84.lo và php82.lo hoặc sử dụng các cổng khác nhau để chuyển hướng lưu lượng đến vhost đúng, và chỉ tạo một vhost cho mỗi phiên bản PHP.

Dự án của bạn sẽ có thể truy cập qua https://php84.lo hoặc http://localhost:8084, cách thức này tùy thuộc vào bạn để thử nghiệm.

Lưu ý rằng 80 là cổng HTTP mặc định và 443 là cổng HTTPS mặc định, mặc dù 8080 thường được sử dụng cho các server phát triển (ví dụ: Vue.js).

MariaDB / MySQL

Trên thực tế không quan trọng lắm việc bạn sử dụng MySQL hay MariaDB trong hầu hết các trường hợp. Có (những) sự khác biệt nhỏ, vì vậy hãy chọn cái mà khách hàng của bạn sử dụng trong sản xuất. Hoặc cài đặt cả hai.

Truy cập MariaDB download page và chọn một phiên bản. Mình khuyên bạn nên chọn một phiên bản “hỗ trợ dài hạn”. Tại thời điểm viết, phiên bản này sẽ là 11.4.

Cài đặt cơ sở dữ liệu vào C:\dev\MariaDB. Mình cũng tạo một thư mục riêng cho dữ liệu của cơ sở dữ liệu, ví dụ: C:\dev\MariaDB_data.

Tệp my.ini của mình nằm trong thư mục MariaDB_data trông như thế này:

[mysqld]
datadir=C:/dev/local/MariaDB_data
port=3306
character-set-server=utf8mb4
innodb_buffer_pool_size=8G
max_heap_table_size=1G
[client]
port=3306
plugin-dir=C:/dev/MariaDB/lib/plugin

Mình thích thiết lập utf8mb4 làm charset mặc định và tăng các giới hạn bộ nhớ để hoạt động mượt mà. Mình không chắc phần [client] có tác dụng gì.

Để quản lý cơ sở dữ liệu, mình sử dụng Adminer, một công cụ tối giản nhưng cung cấp tất cả tiện ích mà mình cần. Bạn có thể xem xét phpMyAdmin hoặc HeidiSQL hay một số công cụ tương tự.

Nếu bạn cần cài đặt nhiều phiên bản, hoặc cả MySQL và MariaDB, hãy điều chỉnh các cổng để mỗi cái lắng nghe trên một cổng khác nhau. Điều này sẽ phải được phản ánh trong ứng dụng của bạn.

Cũng như với Apache, bạn có thể muốn xác nhận cấu hình bằng cách gọi mysqld với tham số --validate-config.

Làm việc với nhiều phiên bản PHP song song

Bây giờ đến phần scripting. Không có vhosts trong CLI, vì vậy nếu bạn mở terminal (CMD, command line) và gõ php -v, bạn sẽ nhận được phiên bản PHP mà bạn đã cài đặt mặc định, nếu có.

Mẹo ở đây là tạo một thư mục (mà mình sẽ gọi là “thư mục bin”) trong C:/dev/bin và đưa nó vào biến PATH hệ thống. Ở đây, mình sẽ đặt các tệp lệnh .bat của mình.

Đi tới “Chỉnh sửa biến môi trường hệ thống” và chỉnh sửa biến PATH sao cho nó bao gồm thư mục này. Windows sẽ chọn nhị phân đầu tiên trong thứ tự mà các thư mục xuất hiện trong biến PATH, vì vậy hãy đặt phần C:/dev/bin ở vị trí phù hợp.

Khởi động một terminal mới và gõ echo %PATH%… nó sẽ hiển thị ở đó (bạn phải khởi động lại terminal sau khi thay đổi biến PATH).

Trong thư mục bin này, mình tạo các tệp như php84.bat cho mỗi phiên bản PHP. Mình cũng tạo com84.bat cho composer chạy với PHP 8.4 và bất kỳ phiên bản nào khác. Bằng cách này, mình có thể gọi các tệp nhị phân PHP một cách chọn lọc.

@echo off

REM PHP 8.4

REM   %* passes all the input arguments to the PHP call
"C:/dev/php/php84/php.exe" %*

@echo off
REM    Composer running using PHP 8.4

REM   %* passes all the input arguments to the PHP call
php84 "C:\dev\composer\composer.phar" %*

Bây giờ mình có thể gọi php84 cho PHP v8.4 và com84 cho Composer chạy trên PHP 8.4. Mình tạo một lối tắt như vậy cho mỗi phiên bản mình cài đặt và có một phiên bản PHP làm mặc định cho php và com/composer.

Scripts Composer

Thông thường, nếu bạn chỉ định một script chạy PHP hoặc lệnh Composer khác trong composer.json như sau:

{
  "scripts": {
    "cache:warmup": ["php bin/cache-warmup.php"],
    "cache:purge": ["php bin/cache-purge.php"],
    "cache:test": ["composer cache:warmup", "composer cache:purge"]
  }
}

Bạn sẽ thấy script được gọi bằng nhị phân php/composer mặc định chứ không phải cái mà bạn chạy lệnh với.

{
  "scripts": {
    "check": ["php -v"]
  }
}

Script này lẽ ra sẽ in ra phiên bản php đúng, nhưng nó lại in ra phiên bản PHP mặc định của hệ thống mỗi lần, điều này không phải là điều mà chúng ta mong muốn:

Mẹo là sử dụng @php và @composer trong tệp JSON, điều này cho phép composer “sử dụng cùng nhị phân PHP như nó đã được gọi bởi”:

{
  "scripts": {
    "check": ["@php -v"]
  }
}

Như một quy tắc chung, luôn chỉ định các script của bạn là @php và @composer:

{
  "scripts": {
    "cache:warmup": ["@php bin/cache-warmup.php"],
    "cache:purge": ["@php bin/cache-purge.php"],
    "cache:test": ["@composer cache:warmup", "@composer cache:purge"]
  }
}

Built-in server

Giả định rằng dự án của bạn sử dụng thư mục /public làm document root, bạn cũng có thể chỉ cần chạy built-in PHP server, nhưng hãy nhớ rằng nó không được thiết kế để phục vụ các tệp tĩnh (hình ảnh, CSS, JS).

Trong terminal, chạy:

php -S 127.0.0.1:42300 -t public/

Mình đang sử dụng cấu trúc composer.json như thế này, sau đó mình gọi nó là composer serve trong thư mục dự án:

{
  "scripts": {
    "serve": [
      "Composer\\Config::disableProcessTimeout",
      "@php -S 127.0.0.1:42300 -t public/"
    ]
  },
  "scripts-descriptions": {
    "serve": "Chạy một server PHP nhẹ"
  }
}

Nhớ rằng, bạn đang kiểm soát phiên bản PHP mà bạn chạy script với. Dù là com84 serve hay com82 serve.

Các script hữu ích khác

Một script để khởi động server Apache (với mô-đun PHP), cơ sở dữ liệu MariaDB và Garnet cho cache - amp-on.bat:

@echo off

REM Khởi động Apache, MariaDB và Garnet
sc start ApacheHTTPServer
sc start MariaDB
sc start GarnetServer

… và script amp-off chứa sc stop để dừng các dịch vụ như vậy.

Sau đó là một script để khởi động lại Apache. Script này phức tạp hơn và chờ dịch vụ dừng và khởi động:

@echo off

REM Khởi động lại Apache
set "ServiceName=ApacheHTTPServer"

REM Dừng dịch vụ
echo Dịch vụ %ServiceName% đang tắt.
sc stop %ServiceName%

REM Chờ dịch vụ dừng
:LOOP
timeout /t 2 /nobreak >nul
sc query %ServiceName% | find "STATE" | find "STOPPED" >nul
if %errorlevel% equ 0 (
    goto START_SERVICE
) else (
    goto LOOP
)

:START_SERVICE
REM Khởi động dịch vụ
echo.
echo Dịch vụ %ServiceName% đang khởi động.
sc start %ServiceName%

REM Chờ dịch vụ khởi động
:WAIT_START
timeout /t 2 /nobreak >nul
sc query %ServiceName% | find "STATE" | find "RUNNING" >nul
if %errorlevel% equ 0 (
    goto SHOW_MESSAGE
) else (
    goto WAIT_START
)

:SHOW_MESSAGE
echo.
echo Dịch vụ %ServiceName% hiện đang chạy.

Kết luận

Đây không phải là một thiết lập ngắn gọn, nhưng nó đáng giá trong dài hạn. Việc cài đặt thêm các phiên bản PHP chỉ cần tạo một vài tệp và thư mục và sao chép-những tệp php.ini, đôi khi điều chỉnh một giá trị ở đây và đó. Cập nhật cũng rất dễ dàng.

Bạn sẽ hưởng lợi từ một cấu hình Windows hiệu quả và linh hoạt. Không còn phải chờ đợi các nhà phát hành bên thứ ba phát hành phiên bản mới!
